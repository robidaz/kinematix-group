# VendorVault — KineMatix Group

A lightweight Angular demo app that consolidates a vendor catalog, contract metadata, and AI-assisted vendor matching into a single tabbed shell.

This is a portfolio demo — it has no backend, no authentication, and no database. All data lives in local JSON, markdown, and text assets. The only outbound HTTP request the app makes is the on-demand call to the Anthropic Claude API when the user clicks **Find Vendors**.

---

## Tech stack

- **Angular 18** (standalone components)
- **Syncfusion Angular** — Tab, Grid, Dialog, AutoComplete, Button, ProgressBar, Toast
- **ngx-markdown** for the email memo and design-requirements pages
- **Anthropic Claude API** (`claude-3-5-sonnet-20241022`) for the vendor matching prompt
- **Express** for serving the production build on Heroku

## Local setup

### 1. Place the Syncfusion license key

The license is read at runtime from a static asset. Put your Syncfusion key in **two** locations (the file at the project root is the source of truth; the asset copy is what the browser fetches):

```
syncfusion_api_key.txt                       # project root
src/assets/config/syncfusion_api_key.txt     # served as a static asset
```

A simple way to keep them in sync:

```bash
cp syncfusion_api_key.txt src/assets/config/syncfusion_api_key.txt
```

### 2. Place the Anthropic API key

Replace the placeholder in `src/assets/config/anthropic_key.txt` with your actual key:

```
src/assets/config/anthropic_key.txt
```

> **Demo-only pattern:** this ships the API key to the browser. Do not use this pattern in production — proxy the API call through a server you control.

If you leave the placeholder in place, the app falls back to a local heuristic filter so the demo still works without a key.

### 3. Generate vendor logos (optional)

VendorVault references partner logos at `assets/logos/<slug>.png`. The repo does not commit these binaries — you generate them locally:

```bash
# Place the saved Dynamix partners HTML at the project root
cp /path/to/exported/dynamix_partners.html ./dynamix_partners.html

pip install -r requirements.txt
python scrape_logos.py
```

The script parses `<img class="company-logo">` tags, downloads each image, and writes normalized PNG files into `src/assets/logos/`. Vendors without a matching local logo fall back to a colored UI Avatars URL automatically.

### 4. Install Node dependencies and run

```bash
npm install
npm run serve     # ng serve, http://localhost:4200
```

## Production build

```bash
npm run build
node server.js    # http://localhost:8080
```

`server.js` is a minimal Express static server pointing at `dist/vendorvault/browser/`.

---

## Heroku deployment

The app is Heroku-ready. From the project root:

```bash
heroku create vendorvault-demo
git push heroku main
```

Heroku will:

1. Run `npm install` (devDependencies included via the buildpack default)
2. Run `npm run heroku-postbuild` → `ng build --configuration production`
3. Boot via `Procfile` → `web: node server.js`

> **Note:** the API key files in `src/assets/config/` are **gitignored**. If you want them in your Heroku build, materialize them at release time from Heroku config vars rather than committing them.

## Notes

- Vendor data is **simulated** for demonstration purposes. Some company names are real partners; all metrics, contract values, and renewal dates are fabricated.
- The AI prompt panel calls Anthropic only on user click — never on page load.
- The grid is a Syncfusion DataGrid with sorting, search (toolbar), column chooser, pagination, and row-click → detail dialog.
